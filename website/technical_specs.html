Especificaciones Técnicas

Detalles técnicos completos sobre la arquitectura, protocolos y rendimiento de DroidMouse.

Arquitectura del Sistema

DroidMouse utiliza una arquitectura cliente-servidor con comunicación TCP/IP:

  • Cliente Android: App nativa en Kotlin que captura datos de sensores
  • Servidor Python: Procesa datos y controla el ratón del sistema
  • Protocolo: TCP/IP con formato JSON para intercambio de datos

Sensores y Frecuencia

  • Giroscopio: Captura rotación en ejes X, Y, Z (grados/segundo)
  • Acelerómetro: Mide aceleración lineal (m/s²)
  • Frecuencia de muestreo: 50Hz (20ms entre muestras)
  • Precisión: 16-bit para giroscopio, 12-bit para acelerómetro

Algoritmos de Procesamiento

  • Filtro de Kalman: Elimina ruido y vibraciones
  • Calibración automática: Compensa deriva del sensor
  • Mapeo de coordenadas: Convierte rotación a movimiento de cursor
  • Suavizado: Interpolación para movimientos fluidos

Rendimiento

  • Latencia: < 2ms (servidor) + latencia de red
  • Precisión: ±0.1° para rotación, ±0.01g para aceleración
  • Consumo de batería: ~5% por hora en uso continuo
  • Uso de memoria: ~15MB en Android, ~25MB en PC

Compatibilidad

Android

  • Android 7.0 (API 24) o superior
  • Giroscopio y acelerómetro requeridos
  • Permisos: SENSORS, INTERNET, ACCESS_NETWORK_STATE

PC (Servidor)

  • Python 3.8+
  • Windows 10+, macOS 10.14+, Linux (kernel 4.0+)
  • Dependencias: pynput, numpy, socket

Protocolo de Comunicación

Los datos se transmiten en formato JSON con la siguiente estructura:

{
  "timestamp": 1640995200.123,
  "gyro": {"x": 0.12, "y": -0.08, "z": 0.05},
  "accel": {"x": 0.01, "y": 0.02, "z": 9.81},
  "status": "connected"
}

Seguridad

  • Comunicación local (no requiere internet)
  • Validación de datos en servidor
  • Límites de movimiento para prevenir comportamientos erráticos
  • Timeout de conexión automático
